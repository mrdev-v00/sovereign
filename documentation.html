Getting Started with Sovereign

Welcome to Sovereign! This guide will help you set up the bot for maximum security and functionality.

Note: Sovereign requires Administrator permissions to function properly. Make sure to grant these when inviting the bot.

Quick Start

1

Invite Sovereign to Your Server

Use the invite link with Administrator permissions. The bot will automatically create necessary roles and channels.

2

Run the Setup Wizard

Use the command wizard to interactively configure all modules.

3

Enable AntiNuke Protection

Run antinuke enable to activate the core security system.

4

Configure Whitelist

Add trusted users with whitelist add @User to prevent false positives.

AntiNuke Commands

The AntiNuke system protects your server from malicious actions and unauthorized changes.

Command Description Example
antinuke Opens control panel to enable/disable AntiNuke antinuke or antinuke enable
antinuke status Shows if AntiNuke is enabled or disabled antinuke status
antinuke enable Enables the AntiNuke module antinuke enable
antinuke disable Disables the AntiNuke module antinuke disable
autobackup <off|seconds> Sets automatic backup interval or disables it autobackup 5 or autobackup off
backup Forces a manual backup of the current server setup backup
whitelist add <user> Adds a trusted user to the whitelist whitelist add @User
whitelist remove <user> Removes a user from the whitelist whitelist remove @User
whitelist list Lists all whitelisted users whitelist list
quarantine Lists all users currently quarantined quarantine
quarantine remove <user> Unquarantines a user manually quarantine remove @User

Security Configuration Guide

Follow this step-by-step guide to make Sovereign fully protect your server.

1

Enable AntiNuke

Run antinuke enable to activate protection.

Sovereign will start monitoring all major events: channel delete/create, role delete/create, webhook spam, mass bans/kicks. If anyone not in whitelist performs these actions, they are quarantined instantly and their actions are reverted.

2

Set AutoBackup Interval

Run autobackup 10 to save your guild structure every 10 seconds.

Saves your entire guild structure (channels, roles, etc.). You can turn it off for large guilds with autobackup off. Manual backup anytime with backup.

3

Whitelist Trusted Users

Add trusted administrators with whitelist add @User.

Only whitelisted users can safely perform critical actions (delete/create channels, roles, etc.). Anyone else doing so triggers quarantine.

4

Use Quarantine System Wisely

Check quarantined users with quarantine and unquarantine with quarantine remove @User.

This removes their restricted role and restores normal access. Use this command carefully to ensure you're not unquarantining a malicious user.

5

Test & Audit Regularly

Run check to perform system diagnostics.

Ensures all modules (AntiNuke, Backup, Whitelist, Quarantine) are loaded correctly and verifies file integrity and permissions.

6

Optional Wizard Setup

Use wizard for guided configuration.

Helps you configure everything in an easy menu: default backup interval, quarantine role setup, whitelist management.

Recommended Secure Configuration

Setting Recommendation
AntiNuke ✅ Enabled
AutoBackup Every 5-10s for small servers, 10-30s for large ones
Whitelist Only 1-2 trusted owners
Quarantine Role Minimal permissions, no send/manage perms
Manual Backup Run daily for safety

Troubleshooting

Common issues and their solutions.

Bot Not Responding

  • Check if the bot has necessary permissions
  • Verify the command prefix is correct
  • Ensure the bot is online and not experiencing downtime

AntiNuke Not Working

  • Run antinuke status to check if it's enabled
  • Verify the bot has Administrator permissions
  • Check if the user performing actions is whitelisted

Backup Issues

  • Ensure there's enough storage space
  • Check file permissions for the backup directory
  • Verify the backup interval is set correctly

If you're still experiencing issues, join our support server for assistance from our team and community.
